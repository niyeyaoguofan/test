# Territory Merge Prototype (React + TypeScript)

A polished MVP for an original single-player strategy-puzzle browser game: global slide controls, merge chains, core structure interactions, territory scoring, and local board expansion.

## Rules (precise MVP)

- Board is 6x6 total; center 4x4 starts active.
- Every turn you choose one global slide direction (`up/down/left/right`).
- Basic tiles merge only with same-level basic tiles.
- `level 4 + level 4` produces a random Core (Beacon / Attractor / Growth), not a higher basic tile.
- Core effects resolve in deterministic order each turn: `Beacon (passive territory radius) -> Attractor (pull one nearby low-level tile by 1 cell) -> Growth (spawn every 3 turns per core)`.
- Territory is controlled by core influence over active cells.
- Expansion unlocks a few adjacent inactive cells at score milestones.
- Turn sequence: slide -> merge -> core effects -> territory update -> expansion check -> random spawn -> game over check.
- Lose when no valid moves remain.

## Hidden ambiguities resolved in this MVP

- **Merge chain policy:** one merge per tile per move lane (standard 2048 behavior).
- **Core randomness fairness:** pseudo-random seeded generator; no weighted bias in v1 (uniform among 3 core types).
- **Attractor target selection:** deterministic tie-breaking by lowest level then nearest tile.
- **Growth spawn scope:** adjacent 4-neighbor active empty cells only.
- **Expansion locality:** only frontier cells adjacent to currently controlled territory are unlocked.

## Core data structures

- `GameState`: full snapshot (board, tiles, score breakdown, turn, controlled map, RNG state, game-over flag).
- `Board`: active/inactive cell map plus `newlyActivated` cells for UI emphasis.
- `Tile`: discriminated union:
  - `BasicTile { kind: 'basic', level }`
  - `CoreTile { kind: 'core', subtype: 'beacon' | 'attractor' | 'growth' }`
- `applyAction(state, direction)`: headless step API for future AI usage.
- `serializeState / deserializeState`: JSON transport hooks.

## Scoring

- Merge score: generated by merges (core creation grants larger bump).
- Territory score: `3 * controlled cell count`.
- Survival score: +2 per valid turn.
- Continuity bonus: based on largest connected controlled region.
- Total = sum of above.

## Future AI Extension Notes

The game engine is already designed for non-UI execution:

1. **Headless stepping**
   - Use `createInitialState(seed)` and `applyAction(state, action)` from `src/game/engine.ts`.
2. **Serialization**
   - Persist episode snapshots with `serializeState` and reload with `deserializeState`.
3. **Gym-like wrapper sketch**
   - `reset(seed) -> observation`
   - `step(action) -> { observation, reward, done, info }`
   - `action_space = Discrete(4)` (up/down/left/right)
   - `observation` can flatten tile planes + active mask + controlled mask + scalar features.
4. **Reward shaping options**
   - Dense reward from score delta.
   - Auxiliary rewards for territory continuity, penalties for congestion.
5. **Determinism for evaluation**
   - Fixed seed and logged RNG state ensure reproducible trajectories.
